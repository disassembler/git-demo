Strategies

  • Commit large blocks of unrelated code (BAD)
  • Commit small changes frequently (GOOD)
  • Work on multiple tasks and tease apart commits (GOOD)
  • Commit large blocks of code in fixed time intervals and tease apart into organized commits (GOOD)

All of the above strategies aside from the top one are valid strategies to use with git. If you prefer to commit small changes frequently, make a change and commit. If you prefer to Work on lots of things at once and deal with committing later, that strategy works too, but become really good friends with git add --patch if this is your style. Or, if you commit in blocks of code in time intervals and break things apart later, that can be done as well. Your tool of choice in that scenario is rebase -i. Remember, you can always alter history until you push.
